# Smoke Checklist — VM Mobile

## Цель

Проверить, что базовые пользовательские сценарии VM Mobile работают без блокирующих дефектов.

## Подготовка

- выполнен `pnpm install`
- выполнен запуск mobile-приложения через Expo
- приложение открывается на устройстве

## Smoke-сценарии

### 1. Экран входа

- [ ] приложение открывается на экране входа
- [ ] поля логина и пароля отображаются корректно
- [ ] при пустых полях вход не выполняется
- [ ] при непустых полях выполняется переход в каталог

### 2. Каталог

- [ ] каталог лекций отображается
- [ ] поиск по строке работает
- [ ] пустой результат поиска отображается корректно
- [ ] карточка последней открытой лекции отображается после повторного открытия каталога
- [ ] оффлайн-блок отображается без ошибок

### 3. Детали лекции

- [ ] карточка лекции открывается из каталога
- [ ] отображаются описание, блоки и требования к участию
- [ ] кнопка возврата в каталог работает
- [ ] кнопка перехода к сессии работает

### 4. Сессия

- [ ] экран сессии открывается
- [ ] отображаются код, статус, подключение, участники и текущий блок
- [ ] переход к заданию работает
- [ ] возврат к лекции работает

### 5. Задание

- [ ] экран задания открывается
- [ ] таймер запускается
- [ ] выбор ответа работает
- [ ] короткий ответ вводится корректно
- [ ] отправка ответа работает
- [ ] при отсутствии ответа показывается ошибка
- [ ] по истечении таймера происходит автоматическая отправка

### 6. Результат

- [ ] экран результата отображается
- [ ] показываются баллы, правильный ответ и пояснение
- [ ] кнопка возврата к сессии работает
- [ ] кнопка завершения и возврата к лекции работает

### 7. Профиль

- [ ] экран профиля открывается
- [ ] отображаются данные пользователя
- [ ] переключение темы работает
- [ ] переключение уведомлений работает
- [ ] выход из аккаунта работает

### 8. Кэш

- [ ] после перезапуска приложения восстанавливается тема
- [ ] после перезапуска приложения восстанавливается последняя открытая лекция
- [ ] каталог доступен из локального кэша

## Критерии прохождения

Smoke-проверка считается пройденной, если:

- нет crash
- нет зависания UI
- основные экраны открываются
- demo flow проходит от входа до результата задания
